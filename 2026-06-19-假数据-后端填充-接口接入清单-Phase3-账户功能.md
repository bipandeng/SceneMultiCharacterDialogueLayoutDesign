# LinguaScene — 假数据 / 后端填充 / 接口接入清单(Phase 3)

> **创建日期**:2026-06-19
> **范围**:Phase 3(账户功能)— 二维码 / 等级 / 提醒 / 收藏 / 下载 / 注销
> **前序依赖**:Phase 1(`/me/preferences/*` 模式可复用)、Phase 2(`user_characters` / `user_scenes` 表)
> **后续**:Phase 4(社交系统)— 二维码扫码后的加好友流程

---

## 1. 目的

把 ProfileView 中仍以"硬编码"形式存在的个人偏好/账户能力接入后端或本地能力,让用户的"我的"页面真正反映其账户状态。

**YAGNI 原则**:Phase 3 不做完整社交系统(留给 Phase 4),只做"账户 + 个人偏好"。二维码在 Phase 3 仍是"展示码",扫码实际行为在 Phase 4 实现。

## 5. 后端 API 详细规格

### 5.1 `GET /me/qrcode`

**用途**:ProfileView 二维码

**返回**:
```json
{
  "qrCodeBase64": "iVBORw0KGgoAAAANSUhEUgAA...",
  "content": "linguascene://user/12345",
  "userId": 12345,
  "displayName": "Alice"
}
```

**说明**:
- `qrCodeBase64` 是后端预渲染的 PNG(base64 编码),iOS 直接显示
- `content` 是扫码后的 deep link,Phase 3 仅展示,Phase 4 实现扫码行为
- Phase 3 不做时效,签名靠 JWT 内嵌到 content

### 5.2 `GET /me/level`

**用途**:ProfileView 等级展示

**返回**:
```json
{
  "currentLevel": "intermediate",
  "nextLevel": "advanced",
  "progressPercent": 65,
  "criteria": {
    "conversations": 23,
    "scenesCompleted": 8,
    "totalMinutes": 156,
    "thresholds": {
      "beginner": { "minConversations": 0 },
      "intermediate": { "minConversations": 10 },
      "advanced": { "minConversations": 30 },
      "native": { "minConversations": 80 }
    }
  }
}
```

**算法**:
```
level = max(满足阈值的最高档)
progressPercent = (current_min_conv - prev_min_conv) / (next_min_conv - prev_min_conv) × 100
```

**Phase 3 不存表**,实时算。

### 5.3 `GET /me/preferences/reminder`

**返回**:
```json
{
  "enabled": true,
  "hour": 19,
  "minute": 0
}
```

### 5.4 `PATCH /me/preferences/reminder`

**请求体**:
```json
{
  "enabled": false,
  "hour": 20,
  "minute": 30
}
```

**校验**:`hour` ∈ [0, 23], `minute` ∈ [0, 59]

**副作用**:iOS 端 PATCH 后,本地 `ReminderService` 同步更新 `UNUserNotificationCenter` 调度

### 5.5 `GET /me/favorites/characters`

**返回**:角色列表(同 `GET /characters`,加 `favoritedAt` 字段)

### 5.6 `POST /me/favorites/characters/{character_id}`

**用途**:收藏 / 取消收藏(toggle)

**副作用**:`favorited = !favorited` 切换

**返回**:当前状态 `{"favorited": true}`

### 5.7 `GET /me/favorites/scenes` / `POST /me/favorites/scenes/{scene_id}`

同 character

### 5.8 `GET /me/downloads/scenes`

**返回**:已下载场景列表(含 `downloadSizeKb`、`downloadedAt`)

### 5.9 `POST /me/downloads/scenes/{scene_id}`

**用途**:下载场景(实际是 iOS 本地缓存 + 后端记录元信息)

**后端行为**:写 `user_scene_downloads` 表(`user_id`, `scene_id`, `downloaded_at`)

**iOS 行为**:把 scene DTO + character DTO 写到本地 JSON 文件

### 5.10 `DELETE /me/downloads/scenes/{scene_id}`

删除本地缓存 + 后端记录

### 5.11 `DELETE /me`

**用途**:注销账号

**后端副作用**:
1. 软删除 `users` 行(`deleted_at = now()`),保留数据 30 天
2. 删除所有用户的 conversations / messages / favorites / downloads / user_scenes / user_characters / user_learned_words

**返回**:`204 No Content`

**iOS 副作用**:删除后调用 `AuthService.shared.logout()`,回到 login 页

## 6. 数据库变更

### 6.1 新建表 `user_favorites_characters`

```sql
CREATE TABLE user_favorites_characters (
    user_id      INTEGER NOT NULL,
    character_id VARCHAR(36) NOT NULL,             -- 系统 character 或用户 character 共用 ID 空间
    favorited_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, character_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE INDEX idx_fav_char_user
    ON user_favorites_characters(user_id, favorited_at DESC);
```

### 6.2 新建表 `user_favorites_scenes`

```sql
CREATE TABLE user_favorites_scenes (
    user_id   INTEGER NOT NULL,
    scene_id  VARCHAR(36) NOT NULL,
    favorited_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, scene_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE INDEX idx_fav_scene_user
    ON user_favorites_scenes(user_id, favorited_at DESC);
```

### 6.3 新建表 `user_scene_downloads`

```sql
CREATE TABLE user_scene_downloads (
    user_id        INTEGER NOT NULL,
    scene_id       VARCHAR(36) NOT NULL,
    downloaded_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    size_kb        INTEGER NOT NULL DEFAULT 0,     -- iOS 端缓存大小上报
    PRIMARY KEY (user_id, scene_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE INDEX idx_downloads_user
    ON user_scene_downloads(user_id, downloaded_at DESC);
```

### 6.4 不修改的表

- `users`(沿用,`deleted_at` 字段如不存在则加)
- Phase 1 / Phase 2 已有表(全部不修改)

### 6.5 可选:`users.deleted_at` 字段

```sql
ALTER TABLE users ADD COLUMN deleted_at DATETIME;
-- 软删除标记,30 天后由定时任务物理删除
```

## 8. 错误处理

### 8.1 后端

| 错误码 | 场景 |
|---|---|
| 401 | JWT 缺失/过期 |
| 403 | 删除他人收藏 / 下载(内部校验) |
| 404 | 资源不存在 |
| 422 | 字段越界(reminder.hour > 23) |
| 500 | SQL 异常 |
| 409 | 注销账号失败(被引用) |

### 8.2 iOS

- 收藏 / 下载 toggle 失败时,UI 回滚到之前状态并 toast
- 提醒时间设置失败时,保留旧 schedule
- 删除账号后强制跳登录页,清空所有本地缓存

```bash
TOKEN="eyJhbGci..."

  -H "Authorization: Bearer $TOKEN" | jq
```

```bash
curl -X PATCH "http://localhost:8000/me/preferences/reminder" \
```

```bash
curl -X POST "http://localhost:8000/me/favorites/characters/{character_id}" \
```

## 11. 算法锚定(关键决策回放)

| 决策点 | 选择 | 锚定理由 |
|---|---|---|
| Q11 二维码用途 | 加好友码(扫码 = 发好友请求) | Phase 3 只展示,Phase 4 触发 |
| Q12 提醒方式 | iOS 本地推送 | 无后端压力,离线也能收到 |
| Q13 等级档位 | 粗 4 档(beginner/intermediate/advanced/native) | 复用现有 difficulty |
| Q14 收藏 + 下载 | 分开两张表 | 业务模型不同(快关系 vs 慢缓存) |
| Q15 注销方式 | 软删除 + 30 天后物理删除 | 给用户反悔机会,符合数据合规 |
