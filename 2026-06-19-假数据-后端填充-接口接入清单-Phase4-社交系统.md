> **创建日期**:2026-06-19
> **范围**:Phase 4(社交系统)— 仅加好友(扫码 + 双向确认)
> **前序依赖**:Phase 3(二维码 `content` 字段已带 user_id,可直接解析)
> **后续**:本 spec 即路线图终点,不做排行榜 / 分享 / 对练

## 1. 目的

完成 ProfileView 二维码的"扫码 → 加好友"闭环,让 LinguaScene 拥有最小可用的社交能力。

**YAGNI 原则**:Phase 4 严格控制在"加好友"这一动作,不做分享 / 排行 / 对练 / 动态信息流。所有超出"加好友"边界的特性都明确延后到未来或不做。

## 5. 后端 API 详细规格

### 5.1 状态机

```
friendships / friend_requests 表:

状态        | 触发                              | 下一状态
pending     | A 发起                             | accepted / rejected / cancelled
accepted    | B 接受                             | (终态,删除走 DELETE)
rejected    | B 拒绝                             | (终态)
cancelled   | A 在 pending 状态撤回              | (终态)
```

**数据模型**:`friend_requests` 一行表示"一次申请",`friendships` 一行表示"已是好友"。两者关系 = accepted 之后 friend_requests 标记 closed,同时写 friendships。

### 5.2 `GET /me/friends`

**用途**:好友列表

**返回**:
```json
{
  "friends": [
    {
      "userId": 67890,
      "displayName": "Bob",
      "avatarInitial": "B",
      "becameFriendsAt": "2026-06-15T10:00:00Z",
      "mutualFriends": 3
    }
  ],
  "total": 12
}
```

**SQL**:
```sql
SELECT
    u.id AS user_id, u.display_name, u.avatar_initial,
    f.created_at AS became_friends_at,
    (SELECT COUNT(*) FROM friendships f2
     WHERE f2.user_id = u.id
       AND f2.friend_id IN (SELECT friend_id FROM friendships WHERE user_id = :me)) AS mutual_friends
FROM friendships f
JOIN users u ON u.id = f.friend_id
WHERE f.user_id = :me
ORDER BY f.created_at DESC;
```

### 5.3 `GET /me/friends/requests?status=pending`

**返回**:入站请求列表(发给我的、状态 pending)

```json
{
  "requests": [
    {
      "requestId": "uuid",
      "fromUserId": 67890,
      "fromDisplayName": "Bob",
      "fromAvatarInitial": "B",
      "message": "Hi, let's practice together!",
      "createdAt": "2026-06-19T10:00:00Z"
    }
  ]
}
```

**请求参数**:
- `status` ∈ `pending` / `accepted` / `rejected` / `cancelled`(默认 `pending`)

### 5.4 `POST /me/friends/requests/{user_id}`

**用途**:发起好友请求

**请求体**:
```json
{
  "message": "Hi, let's practice together!"
}
```

**校验**:
- 不能给自己发(403)
- 已经是好友(409)
- 已经有 pending 请求(409,提示"已发送过请求")

**副作用**:写 `friend_requests` 表,状态 `pending`

**返回**:完整 request 对象

### 5.5 `POST /me/friends/requests/{request_id}/accept`

**用途**:接受好友请求

**前置**:必须是 `pending` 状态 + `to_user_id = me`

**副作用**:
1. `friend_requests.status = accepted`
2. 写 `friendships(user_id = me, friend_id = from_user_id)`
3. 写 `friendships(user_id = from_user_id, friend_id = me)`(双向)

**返回**:`{"status": "accepted"}`

### 5.6 `POST /me/friends/requests/{request_id}/reject`

**用途**:拒绝好友请求

**副作用**:`friend_requests.status = rejected`

**返回**:`{"status": "rejected"}`

### 5.7 `POST /me/friends/requests/{request_id}/cancel`

**用途**:发起方撤回(只能在 pending 状态)

**副作用**:`friend_requests.status = cancelled`

**返回**:`{"status": "cancelled"}`

### 5.8 `DELETE /me/friends/{user_id}`

**用途**:删除好友

**副作用**:删除 `friendships` 两条记录

**返回**:`204 No Content`

### 5.9 `GET /users/{user_id}`

**用途**:用户主页(公开版)

**返回**:
```json
{
  "userId": 67890,
  "displayName": "Bob",
  "avatarInitial": "B",
  "joinedAt": "2026-04-01T00:00:00Z",
  "level": "intermediate",
  "friendsCount": 12,
  "relationship": "friend",
  "mutualFriends": 3
}
```

**`relationship` 枚举**:
- `none`:无关系
- `pending_sent`:我发给他,待他接受
- `pending_received`:他发给我,待我接受
- `friend`:已是好友
- `self`:自己(404 但 iOS 端要识别)

### 5.10 `GET /users/search?q=Bob&limit=20`

**用途**:用户搜索(Phase 4 简化版,只搜 displayName)

**返回**:最多 20 条匹配

**校验**:`q` 长度 ≥ 2,否则 422

## 6. 数据库变更

### 6.1 新建表 `friend_requests`

```sql
CREATE TABLE friend_requests (
    id              VARCHAR(36) PRIMARY KEY,                    -- UUID
    from_user_id    INTEGER NOT NULL,
    to_user_id      INTEGER NOT NULL,
    message         VARCHAR(200),
    status          VARCHAR(20) NOT NULL DEFAULT 'pending',     -- pending / accepted / rejected / cancelled
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(from_user_id, to_user_id, status),                  -- 同一 from→to 只能有一个 pending(防止重复发)
    FOREIGN KEY (from_user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (to_user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE INDEX idx_friend_req_to
    ON friend_requests(to_user_id, status, created_at DESC);
CREATE INDEX idx_friend_req_from
    ON friend_requests(from_user_id, status);
```

### 6.2 新建表 `friendships`

```sql
CREATE TABLE friendships (
    user_id     INTEGER NOT NULL,
    friend_id   INTEGER NOT NULL,
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, friend_id),
    CHECK (user_id != friend_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (friend_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE INDEX idx_friendships_user
    ON friendships(user_id, created_at DESC);
```

### 6.3 不修改的表

- 现有所有表(沿用)

## 8. 错误处理

### 8.1 后端

| 错误码 | 场景 |
|---|---|
| 401 | JWT 缺失/过期 |
| 403 | 给自己发请求 / 接受不是给自己的请求 |
| 404 | 用户 / 请求不存在 |
| 409 | 重复发请求 / 已是好友 |
| 422 | 搜索词 < 2 字 |
| 500 | SQL 异常 |

### 8.2 iOS

- 扫码失败:不合法 content → toast "无效二维码"
- 已是好友时点"加好友":按钮变灰显示"已是好友"
- 接受 / 拒绝请求后 row 立刻从 inbox 消失(乐观更新)
- 网络失败:toast 错误,row 回滚

### 9.1 后端(curl 例子)

**搜索用户**:
```bash
TOKEN_A="eyJhbGci..."  # 用户 A
```

```bash
curl "http://localhost:8000/users/67890" \
  -H "Authorization: Bearer $TOKEN_A" | jq
```

**发起好友请求**:
```bash
curl -X POST "http://localhost:8000/me/friends/requests/67890" \
  -H "Authorization: Bearer $TOKEN_A" \
  -H "Content-Type: application/json" \

```

**用户 B 看 inbox**:
```bash
TOKEN_B="eyJhbGci..."  # 用户 B
curl "http://localhost:8000/me/friends/requests?status=pending" \
  -H "Authorization: Bearer $TOKEN_B" | jq
```

**B 接受请求**:
```bash
curl -X POST "http://localhost:8000/me/friends/requests/{request_id}/accept" \
  -H "Authorization: Bearer $TOKEN_B" | jq
```

**A 查好友列表**:
```bash
curl "http://localhost:8000/me/friends" \
  -H "Authorization: Bearer $TOKEN_A" | jq
# 应看到 B
```

**双向记录验证**:
```sql
-- 数据库直查
SELECT * FROM friendships WHERE user_id = A OR friend_id = A;
```

```bash
# 再次发起同一请求
```


### 9.3 端到端

- 用户 A 打开自己的二维码 → 用 B 的手机扫一下 → B 看到 A 的公开主页 → 点"加好友" + 写消息 → A 在 inbox 看到 → A 接受 → A 好友列表出现 B
- B 删除 A → 双方好友列表都消失
- A 再点 B 的主页 → 按钮变回"加好友"
- 注销账号后,所有发出去的请求 + friendships 自动级联删除(由 FK ON DELETE CASCADE 兜底)
