# LinguaScene — 假数据 / 后端填充 / 接口接入清单

> **创建日期**:2026-06-19
> **范围**:Phase 1(学习统计面板)— HomeView + ProgressView 全部统计项 + ProfileView preferences
> **后续**:Phase 2(用户内容生产) / Phase 3(账户功能) / Phase 4(社交系统) 各自独立 spec

---

## 1. 目的

把当前 UI 中的"假数据"逐步替换为真实后端数据,让用户看到 App 是"活的"。Phase 1 专注于**统计面板**——不需要新建大量表,基于已有的 `conversations + messages + scenes + characters + users` 表做聚合查询。

**YAGNI 原则**:Phase 1 不做自定义内容、不做社交、不做等级系统。只做"统计面板从假变真"。

## 8. 后端 API 详细规格

### 8.1 `GET /me/stats/summary`

**用途**:HomeView streakCard + ProgressView 4 项核心指标

**返回**:
```json
{
  "currentStreak": 7,
  "longestStreak": 12,
  "totalConversations": 23,
  "totalMinutesSpent": 156,
  "wordsLearned": 89,
  "scenesCompleted": 8,
  "fluencyScore": 72,
  "lastActiveDate": "2026-06-19T14:23:11Z"
}
```

**SQL 实现思路**(每个字段独立算):
```sql
-- currentStreak: 从今天往前数,连续有对话的天数
-- longestStreak: 历史最长连续天数
-- totalConversations: COUNT(conversations WHERE user_id = ?)
-- totalMinutesSpent: SUM(duration_minutes)
-- wordsLearned: COUNT(user_learned_words WHERE user_id = ?)
-- scenesCompleted: COUNT(DISTINCT scene_id WHERE message_count >= 10)
-- fluencyScore: 5 维平均(发音/听力暂 50)
```

**JWT**:需要 `Bearer <token>` 头部

### 8.2 `GET /me/stats/today`

**用途**:HomeView greeting 的"今日目标 72%" + ProgressView 顶部进度条

**返回**:
```json
{
  "date": "2026-06-19",
  "targetMinutes": 15,
  "completedMinutes": 11,
  "targetConversations": 3,
  "completedConversations": 2,
  "progressPercent": 73,
  "isCompleted": false
}
```

**SQL**:
```sql
SELECT
    -- targetMinutes / targetConversations: 从 user preferences 读
    -- completedMinutes: SUM(duration_minutes) WHERE started_at >= today_start
    -- completedConversations: COUNT(*) WHERE started_at >= today_start
```

### 8.3 `GET /me/stats/calendar?weeks=4`

**用途**:ProgressView 学习日历(4 周 × 7 天 = 28 格热力图)

**查询参数**:`weeks`(默认 4,最大 12)

**返回**:
```json
{
  "weeks": 4,
  "days": [
    { "date": "2026-05-23", "active": true, "conversationCount": 2 },
    { "date": "2026-05-24", "active": false, "conversationCount": 0 },
    ...
  ]
}
```

**判定逻辑(Q3 决策 Y)**:**当日 conversation 数 ≥ 1 即为 active**

**SQL**:
```sql
SELECT
    DATE(started_at, 'localtime') AS study_date,
    COUNT(*) AS conv_count
FROM conversations
WHERE user_id = :user_id
  AND started_at >= datetime('now', '-28 days')
GROUP BY study_date
ORDER BY study_date;
```

**时区**:后端用 SQLite `localtime`(依赖服务器时区设置);iOS 直接接收字符串,不参与日期计算。

### 8.4 `GET /me/stats/fluency`

**用途**:ProgressView 流利度 5 维分析

**返回**:
```json
{
  "pronunciation": 50,
  "vocabulary": 72,
  "grammar": 58,
  "fluency": 55,
  "listening": 50,
  "overall": 61,
  "basedOnMessages": 87
}
```

**算法(Q1 决策 Y — 真实计算,简单加权)**:

```
input: 该用户最近 100 条 message.user
output: 5 个维度分数(0-100)

pronunciation = 50  (Phase 1 兜底,无语音数据)
listening     = 50  (Phase 1 兜底,无语音数据)

vocabulary    = min(100, unique_word_count / message_count × 100 × 1.5)
               简单词频比 + 系数,粗略体现"用词多样性"

grammar       = max(0, (1 - correction_rate) × 100)
               correction_rate = is_corrected=true 的 user 消息数 / 总 user 消息数

fluency       = min(100, avg_words_per_user_message × 8)
               平均句长 × 8,封顶 100

overall       = (pronunciation + vocabulary + grammar + fluency + listening) / 5
```

**SQL**:
```sql
-- 统计 user 消息
SELECT
    text,
    is_corrected,
    (LENGTH(text) - LENGTH(REPLACE(text, ' ', '')) + 1) AS word_count
FROM messages
WHERE conversation_id IN (SELECT id FROM conversations WHERE user_id = :user_id)
  AND sender = 'user'
ORDER BY timestamp DESC
LIMIT 100;
```

**iOS 端**:`pronunciation` 和 `listening` 等于 50 时,旁边加标签 `progress.fluency.noData`("暂无语音数据")。

### 8.5 `GET /me/stats/recent-words?limit=5`

**用途**:ProgressView "最近掌握的单词"

**查询参数**:`limit`(默认 5,最大 20)

**返回**:
```json
{
  "words": [
    { "word": "oat", "firstSeenAt": "2026-06-18T10:23:00Z", "conversationId": 42 },
    { "word": "recommend", "firstSeenAt": "2026-06-17T15:42:00Z", "conversationId": 38 },
    ...
  ]
}
```

**数据源**:`user_learned_words` 表(Q2 决策 Y)

**SQL**:
```sql
SELECT word, first_seen_at, conversation_id
FROM user_learned_words
WHERE user_id = :user_id
ORDER BY first_seen_at DESC
LIMIT :limit;
```

### 8.6 `GET /me/conversations?limit=20&offset=0`

**用途**:ProgressView "历史对话"列表

**查询参数**:
- `limit`(默认 20,最大 50)
- `offset`(默认 0,分页)

**返回**:
```json
{
  "total": 23,
  "conversations": [
    {
      "id": 42,
      "sceneId": "uuid",
      "sceneTitle": "咖啡馆点单",
      "characterName": "Emma",
      "characterAvatar": "👩‍🍳",
      "durationMinutes": 8,
      "messageCount": 14,
      "fluencyScore": 75,
      "startedAt": "2026-06-19T10:23:00Z",
      "updatedAt": "2026-06-19T10:31:00Z"
    },
    ...
  ]
}
```

**`fluencyScore` 字段**:Phase 1 用 conversation 维度的简化分数(取该 conversation 的 user 消息 grammar_score 平均值)。Phase 3 再做详情页。

**SQL**:
```sql
SELECT
    c.id, c.scene_id, c.started_at, c.updated_at, c.duration_minutes,
    s.title AS scene_title,
    ch.name AS character_name,
    ch.avatar_emoji AS character_avatar,
    COUNT(m.id) AS message_count,
    AVG(CASE WHEN m.sender = 'user' AND m.is_corrected = 1 THEN 0 ELSE 1 END) * 100 AS fluency_score
FROM conversations c
JOIN scenes s ON s.id = c.scene_id
JOIN characters ch ON ch.id = c.character_id
LEFT JOIN messages m ON m.conversation_id = c.id
WHERE c.user_id = :user_id
GROUP BY c.id
ORDER BY c.started_at DESC
LIMIT :limit OFFSET :offset;
```

## 9. 数据库变更

### 9.1 新建表 `user_learned_words`(Q2 决策 Y)

```sql
CREATE TABLE user_learned_words (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id         INTEGER NOT NULL,
    conversation_id INTEGER NOT NULL,
    word            VARCHAR(64) NOT NULL,
    first_seen_at   DATETIME NOT NULL,
    UNIQUE(user_id, word),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (conversation_id) REFERENCES conversations(id) ON DELETE CASCADE
);

CREATE INDEX idx_learned_words_user
    ON user_learned_words(user_id, first_seen_at DESC);
```

### 9.2 不修改的表

- `users`(已有字段够用)
- `conversations`(Phase 1 不加字段,`fluency_score` 在响应时实时算)
- `messages`(已有 `is_corrected` 字段够用)
- `scenes` / `characters`(只读)

## 12. 错误处理

### 12.1 后端

| 错误码 | 场景 | 响应体 |
|---|---|---|
| 401 | JWT 缺失/过期 | `{ "error": "unauthorized" }` |
| 422 | 参数越界(如 targetMinutes > 120) | `{ "error": "validation_failed", "field": "targetMinutes" }` |
| 500 | SQL 异常 | `{ "error": "internal_error" }`,日志记录 |

### 12.2 iOS

- 所有 `StatsService` 方法用 `throws`,失败抛 `StatsError.network` / `.decoding` / `.unauthorized`
- ViewModel 接 `catch` 后显示**兜底数据**(保留旧 SampleData 5 秒,然后显示错误 toast)
- 不弹错误弹窗打扰用户(统计类容错)
