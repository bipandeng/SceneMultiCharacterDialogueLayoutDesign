## 1. 目的

让用户能**自己创建**场景和角色,而不是只能消费预置内容。同时执行"场景化对话统一入口"的产品决策:

1. **删除 Characters tab**(场景化后冗余,当前是死路径)— 5 tabs → 4 tabs
2. **统一 ScenePreviewView 入口** — 删除"单角色直进"和"多角色"两个 CTA,合并为 1 个内联的 `CharacterSlotPickerView`,1-3 槽位
3. **用户自创角色** — 主路径在 Profile → "我的 AI 角色",快捷入口在 `CharacterSlotPickerView` 内"+ 自创角色"
4. **升级 `Conversation.characterId` → `characterIds[]`** — 对齐后端多角色流式

**YAGNI 原则**:Phase 2 不做社区、不做公开场景、不做审核、不做模板市场、不做编辑历史、不做 V3 多角色同时发言。

## 5. ScenePreviewView 入口重构

### 5.1 当前架构(已删除)

```
ScenePreviewView
  ├─ startCTA      → Route.chat(scene, [character])          // 单角色直进
  └─ groupChatCTA  → GroupChatPickerView sheet              // 多角色
```

### 5.2 Phase 2 重构后

```
ScenePreviewView
  └─ [开始对话] CTA
        └─ 内联展开 CharacterSlotPickerView(1-3 槽位)
              ├─ 槽位 1:[场景默认推荐角色] / [+] 添加
              ├─ 槽位 2:[James] / [+] 添加
              ├─ 槽位 3:[空] / [+] 添加
              │
              └─ 角色池分组:
                    ├─ 系统角色(可勾选)
                    ├─ 我的角色(自创,可勾选)
                    └─ [+] 创建新角色 → CharacterCreatorView(4 步表单)
              │
              └─ [开始对话] → Route.chat(scene, [ch1, ch2, ch3])
```

### 5.3 `CharacterSlotPickerView` 行为

- 场景默认推荐角色(从 `scene.recommendedCharacterIds.first`)自动填入第 1 槽位
- 用户可删除槽位 / 添加槽位(最多 3)
- "+ 创建新角色"内联 sheet 弹出 `CharacterCreatorView`,创建成功后自动填入下一个空槽位
- 槽位为 0 时 CTA 灰
- 槽位 1 → "开始对话(1 人)",槽位 2/3 → "开始群聊(N 人)"

### 5.4 `CharacterCreatorView`(4 步表单)

```
Step 1 - 基础:
  - 名字 (TextField)
  - emoji (EmojiPicker)
  - 头像色调 (3 选 1: terracotta / olive / stone)

Step 2 - 出身:
  - 出身地 (TextField)
  - 口音 (Picker: 美式 / 英式 / 澳式 / 其他)

Step 3 - 性格:
  - traits chips(从 12 个预设选 2-4 个:patient / warm / strict / humorous ...)

Step 4 - 详细:
  - bio (TextArea,默认折叠)
  - greeting (TextArea,必填)
  - system_prompt 高级编辑(默认折叠,只展示"使用默认模板")
```

**system_prompt 限制**:违禁词列表 + 长度 ≤ 2000 字(MVP 不做内容审核引擎)

### 5.5 `MyCharactersView`(Profile 下的角色管理)

```
我的 AI 角色
─────────────────
[+] 创建新角色
─────────────────
😊 Marcus 老友
   北京 / 美式英语
   性格:warm, humorous
   [编辑]  [删除]
─────────────────
🎓 Ms. Liu
   ...
```

## 7. 后端 API 详细规格

### 7.1 `GET /me/scenes`

**用途**:Profile / Home 显示用户自建的场景

**返回**:
```json
{
  "scenes": [
    {
      "id": "uuid",
      "title": "周末晨跑",
      "summary": "练习和跑友打招呼、约下一次跑步",
      "objectives": ["打招呼", "聊跑步计划", "约下一次"],
      "createdAt": "2026-06-19T10:00:00Z",
      "updatedAt": "2026-06-19T10:00:00Z"
    }
  ]
}
```

### 7.2 `POST /me/scenes`

**请求体**:
```json
{
  "title": "周末晨跑",
  "summary": "...",
  "objectives": ["打招呼", "聊跑步计划"]
}
```

**校验**:
- `title` 长度 1-60
- `summary` 长度 1-500
- `objectives` 长度 1-10,每条 1-100

**返回**:7.1 的 scene 对象(带生成的 id)

### 7.3 `PATCH /me/scenes/{id}`

**请求体**:同 7.2,所有字段可选(部分更新)

**权限校验**:必须是创建者,否则 403

### 7.4 `DELETE /me/scenes/{id}`

**副作用**:`conversations` 表中引用该 scene 的记录保留(仅 `scene_id` 指向已删除场景),UI 渲染时 fallback 显示"已删除场景"

**返回**:`204 No Content`

### 7.5 `GET /me/characters`

**用途**:用户自建角色列表(供 CharacterSlotPickerView 角色池的"我的角色"分组 + MyCharactersView)

**返回**:
```json
{
  "characters": [
    {
      "id": "uuid",
      "name": "李雷",
      "avatarEmoji": "👨‍🎓",
      "avatarColor": "terracotta",
      "subtitle": "中国留学生",
      "origin": "北京",
      "accent": "中式英语",
      "difficulty": "intermediate",
      "traits": ["patient", "humorous"],
      "topics": ["校园生活", "中国美食"],
      "bio": "...",
      "greeting": "Hi! I'm Li Lei, a student from Beijing.",
      "systemPrompt": null,           // 仅 owner 自己看得到
      "createdAt": "2026-06-19T10:00:00Z",
      "updatedAt": "2026-06-19T10:00:00Z"
    }
  ]
}
```

### 7.6 `GET /me/characters/{id}`

**用途**:用户自创角色详情(只 owner 可看 `systemPrompt` 字段)

**权限**:必须是 owner,否则 403

### 7.7 `POST /me/characters`

**请求体**:
```json
{
  "name": "李雷",
  "avatarEmoji": "👨‍🎓",
  "avatarColor": "terracotta",
  "origin": "北京",
  "accent": "中式英语",
  "difficulty": "intermediate",
  "traits": ["patient", "humorous"],
  "topics": ["校园生活", "中国美食"],
  "bio": "...",
  "greeting": "Hi! I'm Li Lei, a student from Beijing.",
  "systemPrompt": null
}
```

**校验**:
- `name` 1-100
- `avatarEmoji` 单个 emoji(后端正则)
- `avatarColor` ∈ `terracotta` / `olive` / `stone`
- `origin` / `accent` ≤ 100
- `difficulty` ∈ enum
- `traits` 长度 2-4,每条 ∈ enum(12 选)
- `topics` 长度 1-8,每条 1-20
- `bio` ≤ 1000
- `greeting` 1-500 必填
- `systemPrompt` ≤ 2000 字 + 违禁词列表校验(违禁词 → 422)

### 7.8 `PATCH /me/characters/{id}`

**请求体**:同 7.7,所有字段可选(部分更新)

**权限**:必须是 owner,否则 403

### 7.9 `DELETE /me/characters/{id}`

**用途**:物理删除(仅 owner)

**副作用**:
- `conversation_participants` 表中该角色引用**软保留**(不级联删除对话),UI 显示"[已删除角色]"
- 后端启用 RLS,非 owner 调此接口直接 403

**返回**:`204 No Content`

### 7.10 `GET /scenes`(Phase 1 已有,Phase 2 扩展)

**新增字段**:
```json
{
  "isUserCreated": false,
  "createdBy": null
}
```

**当 `isUserCreated = true`**:返回 `createdBy = { id, displayName }`

**排序规则**:沿用 Phase 1,系统 / 用户分 tab 切换

### 7.11 `GET /characters`(Phase 1 已有,Phase 2 扩展)

**新增字段**:
- `isUserCreated: bool` — 来自 `user_characters` 表合并
- `isUserCreatable: bool` — 系统角色的"是否可被用户用作模板"标记
- `createdBy: { id, displayName } | null` — `isUserCreated=true` 时返回

**列表组织**:
- `GET /characters` → 系统角色(全量,Phase 1 已有接口,Phase 2 仅扩展字段)
- `GET /me/characters` → 用户自建角色(独立接口)
- CharacterSlotPickerView 角色池分组显示"系统" + "我的"

## 8. 数据库变更

### 8.1 新建表 `user_scenes`

```sql
CREATE TABLE user_scenes (
    id              VARCHAR(36) PRIMARY KEY,
    user_id         INTEGER NOT NULL,
    title           VARCHAR(60) NOT NULL,
    summary         VARCHAR(500) NOT NULL,
    objectives_json JSON NOT NULL,
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE INDEX idx_user_scenes_user
    ON user_scenes(user_id, created_at DESC);

-- RLS
ALTER TABLE user_scenes ENABLE ROW LEVEL SECURITY;
CREATE POLICY user_scenes_isolation ON user_scenes
USING (user_id = current_setting('app.current_user_id')::INTEGER);
```

### 8.2 新建表 `user_characters`

```sql
CREATE TABLE user_characters (
    id              VARCHAR(36) PRIMARY KEY,
    user_id         INTEGER NOT NULL,
    name            VARCHAR(100) NOT NULL,
    avatar_emoji    VARCHAR(16) NOT NULL,
    avatar_color    VARCHAR(20) NOT NULL DEFAULT 'terracotta',
    origin          VARCHAR(100),
    accent          VARCHAR(100),
    difficulty      VARCHAR(20) NOT NULL,
    traits_json     JSON NOT NULL,                       -- ["patient", "humorous"]
    topics_json     JSON NOT NULL,
    bio             TEXT,
    greeting        VARCHAR(500) NOT NULL,
    system_prompt   TEXT,                                -- 用户自填,后端审核
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE INDEX idx_user_characters_user
    ON user_characters(user_id, created_at DESC);

-- RLS
ALTER TABLE user_characters ENABLE ROW LEVEL SECURITY;
CREATE POLICY user_characters_isolation ON user_characters
USING (user_id = current_setting('app.current_user_id')::INTEGER);
```

### 8.3 修改表 `characters`(系统角色,扩展字段)

```sql
ALTER TABLE characters ADD COLUMN is_user_creatable BOOLEAN DEFAULT TRUE;
-- TRUE = 用户可以用作"创建新角色的模板",FALSE = 仅系统使用(如限定场景的"老板"等)
```

**默认值**:`TRUE`,对现有 6 个系统角色全部标记为可创建(产品决策)

### 8.4 修改表 `conversations`(升级为多角色)

```sql
-- 升级:character_id 单值 → character_ids 数组
ALTER TABLE conversations ADD COLUMN character_ids TEXT[] DEFAULT '{}';
UPDATE conversations SET character_ids = ARRAY[character_id::TEXT];
ALTER TABLE conversations ALTER COLUMN character_ids SET NOT NULL;

-- 启用 conversation_participants(后端表已建,业务层启用)
```

**iOS 兼容**:读取时降级 `characterIds.first ?? characterId`,旧对话不丢

### 8.5 不修改的表

- `users`(沿用)
- `messages`(沿用)
- Phase 1 的 `user_learned_words`

## 10. 风险与缓解(采纳已批准文件的 8 项)

| 风险 | 影响 | 缓解 |
|---|---|---|
| 自创角色 `system_prompt` 质量不可控 | LLM 输出混乱 | 模板填空为主,高级用户才开放编辑 |
| 内容审核缺失 | 用户输入违规 persona | MVP:基础违禁词列表 + 长度限制(≤2000 字) |
| 删除用户角色时对话数据孤儿化 | 对话显示已删除角色 | `conversation_participants` 软引用,UI 显示"[已删除]" |
| 群聊 LLM 协调困难 | 多角色相互打断 | V2 严格单角色轮流;V3 引入并行 |
| iOS `Conversation.characterId` → `characterIds` 升级 | 旧对话数据兼容 | 读取时降级 `characterIds.first ?? characterId` |
| `GroupChatPickerView` 整文件删除 | 编译报错 | 同步删 `HomeView.swift:357-361` 的 `RouteScenePreviewCallbacks` 死代码 |
| 现有 `AutoTestRunner.swift:199` UI 测试依赖 group chat | 测试失败 | 同步更新测试,适配 `CharacterSlotPickerView` |
| `Route.chat(scene:, characters:)` 已接数组,签名无需变 | — | 0 改动,反而清爽 |

## 11. 错误处理

### 11.1 后端

| 错误码 | 场景 |
|---|---|
| 401 | JWT 缺失/过期 |
| 403 | 非 owner 编辑/删除他人内容(RLS 兜底) |
| 404 | 资源不存在 |
| 422 | 字段校验失败 / system_prompt 违禁词 |
| 500 | SQL 异常 |

### 11.2 iOS

- `UserContentService` 所有方法用 `throws`
- 编辑器 ViewModel 用 `isSubmitting + errorMessage` 显示内联错误
- 删除带二次确认 sheet(`DeleteConfirmationSheet`)
- 网络失败时保留草稿,提示"稍后重试"
- system_prompt 违禁词时显示"包含违规内容,请修改"内联错误

```bash
curl -X POST "http://localhost:8000/me/characters" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "李雷",
    "avatarEmoji": "👨‍🎓",
    "avatarColor": "terracotta",
    "origin": "北京",
    "accent": "中式英语",
    "difficulty": "intermediate",
    "traits": ["patient", "humorous"],
    "topics": ["校园生活", "中国美食"],
    "bio": "...",
    "greeting": "Hi! I am Li Lei.",
    "systemPrompt": "..."
  }' | jq
```

**编辑角色**:
```bash
curl -X PATCH "http://localhost:8000/me/characters/{id}" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "李雷 2"}' | jq
```

**删除角色**:
```bash
curl -X DELETE "http://localhost:8000/me/characters/{id}" \
  -H "Authorization: Bearer $TOKEN" -i
# 期望 204
```

**列表合并(系统角色含 is_user_creatable 字段)**:
```bash
curl "http://localhost:8000/characters" \
  -H "Authorization: Bearer $TOKEN" | jq '.characters[0]'
# 应包含 isUserCreatable: true, isUserCreated: false
```

**校验 422**:`system_prompt` 超 2000 字
```bash
LONG_PROMPT=$(printf 'a%.0s' {1..2500})
curl -X POST "http://localhost:8000/me/characters" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{...,\"systemPrompt\":\"$LONG_PROMPT\"}" -i
# 期望 422
```

**校验 403**:用户 B 删用户 A 的角色
```bash
# 1. 用户 A 创建角色,记录 id
# 2. 用户 B 登录拿 token
# 3. 用 B 的 token DELETE A 的角色
# 期望 403
```

**RLS 验证**(数据库直查):
```sql
-- 切到 B 的 user context
SET app.current_user_id = B;
SELECT * FROM user_characters;  -- 应只能看到 B 的
SET app.current_user_id = A;
SELECT * FROM user_characters;  -- 应只能看到 A 的
```

### 12.3 端到端

- 创建场景 → 切到"我的"tab → 看到新场景 → 点开 → 编辑 → 保存 → 列表更新
- 创建角色(走 4 步表单) → 在 MyCharactersView 看到 → 进 CharacterSlotPickerView 角色池"我的角色"分组看到 → 选中 → 进 chat → AI 用新角色的 greeting 打招呼
- 走 CharacterSlotPickerView 内"+ 自创角色"快捷入口 → 创建后自动填到下一个空槽
- 删除角色 → CharacterSlotPickerView 角色池"我的角色"分组移除 → 之前用该角色的 conversation 仍能打开(用快照,UI 显示"[已删除角色]")
- 场景对话 1 个角色 vs 3 个角色 → ChatView 多人气泡切换

## 14. 算法锚定(关键决策回放)

| 决策点 | 选择 | 锚定理由 |
|---|---|---|
| Q5 可见性 | 仅自己可见 | Phase 2 不做社区 |
| Q6 场景字段 | title / summary / objectives 必填 | 最小可用,3 字段够用 |
| Q7 角色字段 | name / avatarEmoji / avatarColor / origin / accent / difficulty / traits / topics / bio / greeting / system_prompt | 4 步表单,匹配已批准文件 |
| Q8 场景编辑/删除 | 可编辑可删除 | 用户体验优先,后端加 PATCH |
| Q9 入口重构 | 现在就跟 Phase 2 一起做 | 避免未来双重重构 |
| Q10 列表组织 | 系统 / 我的分 tab | 简单,避免排序复杂度 |
| Q11 Characters tab | 删除 | 场景化后冗余,死路径 |
| Q12 角色管理路径 | Profile → "我的 AI 角色" + 场景选择器内快捷入口 | 双路径,主路径在 Profile |
| Q13 角色表单 | 4 步表单 | 降低单步认知负担 |
| Q14 system_prompt | 用户可填,违禁词 + 2000 字限制 | MVP 不做内容审核引擎 |
| Q15 删除策略 | 物理删除 + conversation_participants 软引用 | 数据干净 + 对话不孤儿化 |
| Q16 角色 system_prompt 可见性 | 仅 owner 自己看得到 | 隐私 |
